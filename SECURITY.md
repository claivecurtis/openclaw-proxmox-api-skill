# Security Policy

## Reporting Vulnerabilities

If you discover a security vulnerability in this project, please report it privately to avoid exposing other users to risk.

You can report vulnerabilities by:

- Emailing [security@claivecurtis.com](mailto:security@claivecurtis.com)
- Creating a private issue on GitHub (if available)

Please include as much detail as possible, including steps to reproduce the vulnerability.

## Threat Model

This project is a skill for interacting with Proxmox API. Potential threats include:

- Unauthorized access to Proxmox servers
- API key exposure
- Injection attacks via API calls
- Data leakage in responses

We strive to mitigate these through secure API handling, input validation, and minimal privilege principles.

## Response

We will acknowledge receipt of your report within 48 hours and provide a more detailed response within 7 days indicating our next steps.

We will keep you informed of our progress and may request additional information from you.

Once the issue is resolved, we will publicly disclose the vulnerability after providing you an opportunity to review the disclosure.
